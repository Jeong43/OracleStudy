2조_후기


김진희 : 오라클 세미 프로젝트를 수행하면서 요구분석서를 꼼꼼히 읽으면서 논리적으로 erd를 작성할 때 동사는 테이블명 작성, 명사는 컬럼으로 작성하는 것을 알게 되었습니다. erd를 작성 후에 물리적인 설계에서 수시로 update화 되는것을 컬럼화 시키면 안된다는 것을 프로젝트 내에서 알게 되었습니다.  수업시간에 이해가 부족했던 프로시저 작성법과 join을 하는 과정에서 join을 어떤 테이블과 연결을 해야지 조인이 되는지 명확하게 구분이 안되서 어려움이 있었습니다. 
하지만 프로젝트 내에서 팀원들이 알려주셔서 제가 해야하는 할당의 부분을 잘 수행 할 수 있게 되었습니다! 
손다정 : 오라클을 배울 때에는 정말 재미있었는데 프로젝트를 하면서 오라클이 너무 어려웠다.
어떻게 해야될지 감도 안 잡히는 상황에서 조원분들 덕분에 잘 따라갈 수 있었다.
특히 연결성이 너무 어려웠고 뷰랑 프로시저도 어려웠는데 프로젝트를 하며 많이 배운 것 같다.
조원 분들 너무 착하시고 모르는 것이 있을 때 다 같이 풀어가려는 모습이 너무 좋았다.
특히 리더분이 너무 리더쉽도 좋고 프로젝트 하면서 막히는 부분이 있을 때 먼저 나서서 해결하려고하고
분위기가 살짝 다운되었을 때에 힘을 불어 넣어 줄 정도로 정말 최고의 팀장이라고 생각한다.
조원들도 좋았고 여러모로 오라클의 애매한 부분을 좀 더 정확하게 알게 된 좋은 기회였다고 생각한다.

윤유동 : 쿼리문작성이 자바보다 쉽다고 생각했던 것을 백번 취소하면서 진행했다. 
ERD 진행을 하면서 DB논리설계를 조금 이해했다고 생각했는데 아니었다. 
모든 진행상황을 파악하는데 DB 관계가 기초가 되어있어야 해서 프로젝트 진행하면서 중요성을 절실히 느꼈다.
DB논리설계는 아직도 잘 모르겠어서 더 많이 공부해야 할 것 같다. 
처음에 필요한 프로시저, 뷰, 함수를 파악한 후에 어디부터 손을 대야할지 감이 안와서 일단 덩어리가 작고,
연관성이 적어보이는 부분 위주로 만들어 나가기 시작했다. 프로시저의 양이 많다보니 프로시저부터 만들었는데 나중에와서 보니 비효율적이었다.
그 전에 프로시저, 뷰, 함수의 연관성을 좀 더 분석해서 생성순서를 정했다면 다들 덜 고생했을텐데 이 부분이 아쉽다.
전체적으로 어느 부분부터 만들어 나갈지, 어느 부분을 만들지 분배하는 건 전적으로 내가 했기때문에 모두를 고생시킨것 같아서 미안하다.
돌이켜보니 비효율이었고, 유능한 팀장은 아니었던거 같은데 서로 잘한다 잘한다 해주는 팀원들 덕분에 제 시간에 결과물이 잘 나온것 같다.

이찬호 : 이번 오라클 세미프로젝트를 하면서 요구 분석서 내용을 바탕으로 ERD논리설계부터 물리설계까지 진행하면서 많은 시행착오가 있었습니다. 
그 중 RED논리설계과정에서 정규화를 통해 테이블을 나누고 제약조건을 설정하는 부분이 이번 프로젝트를 진행하는데 가장 중요했던 부분이라고 생각했습니다. 
부모 테이블과 자식 테이블의 관계 제약조건 파악을 확실히 하지 않고 물리 설계로 넘어가 프로시저와 함수들을 사용하여 
데이터를 구현하는데 많이 어려움을 느꼈고 팀원들과의 소통에 많은 시간을 사용해 진행에 원활하지 못한 부분을 이번 실습을 통해 경험하여 
다음 프로젝트에는 논리설계 과정부터 각 테이블의 관계와 제약조건 이해를 확실히 한 후 진행하여 작업에 문제가 발생하지 않도록 시행할 것입니다.


장민지 : 이번에 오라클이 끝나고 또 미니 프로젝트를 하게되었는데 erd 다이어그램을 그리는 것이 만만한 것이 아니었다
요구사항만 보고 어떤 테이블이 필요한지는 물론 정규화도 해줘야 했는데 너무 어려웠다.
그래서 erd 구성 과정에서 조금 오래 걸렸던 것 같다.. 처음에 필요한 테이블 구성은 된 것 같은데 하다가도 정규화가 안 된 부분들이 있었고,
또 컬럼으로 만들면 안되는 부분들을 컬럼화하여 선생님한테 피드백 받기도 했다
무려 5차 피드백을 걸쳐서 완성..!! 테이블만들고 자료만 입력하면 되는 줄 알았는데 물리적 설계, 필요한 프로시저,뷰 등등 까지
작성하는 거여서 멘붕이 있었지만, erd 작성 후에는 비교적(?) 수월하게 진행 되었다.
먼저 테이블 뼈대를 만들고, 필요한 제약 사항을 추가해놓고나서 필요한 프로시저들을 하나씩 작성하기 시작했는데, 지금 생각해보면
함수를 일찍 만들었으면 프로시저에서 예외처리를 할 때 더욱 수월 했을 것 같다!
프로시저를 만들면서 좀 후반에 함수와 뷰 등등을 작성했는데 그것들을 사용하니까 더 쉬웠다.
또, 팀장님이 중간중간 진행 상황과 우리가 해야할 것들을 수시로 체크해주셨고 한 것들을 종합해주셔서 더욱 빠르게 진행된 것 같다.!
팀원들과 함께하지 못했다면 하지 저어어얼대 못 했을 것 같은 프로젝트였다. 강사님과 2조께 너무너무 감사합니다!!
